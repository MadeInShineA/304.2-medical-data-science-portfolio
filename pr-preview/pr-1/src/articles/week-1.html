Development environment setup

I setup a basic development environment including a project repository (see github)

This includes:

  • a basic README file,
  • a small project website made with quarto deployed to a github page through a github action (see action)

I also convinced my classmates to use github policies as follow:

  • Each feature must be developed on a separate branch
  • A pull request needs to be made to evaluate a feature
  • The feature pull requests need to be verified by at least 2 other individuals, including 1 from another team

Collaborative environment setup

I convinced my fellow colleagues to use Jira. This decision was motivated by the fact that’s it’s an industry standard.

What skills does this demonstrate

Data Engineering Competencies

  • CI/CD & Automation: Deployed GitHub Actions workflow for Quarto → GitHub Pages publishing.
  • Git Workflow Design: Implemented feature-branch strategy with mandatory cross-team PR reviews.
  • Documentation-as-Code: Built living docs (README + Quarto site) for onboarding and reference.

Computer Science Engineering Competencies

  • Toolchain Selection: Introduced Jira, GitHub, and Quarto as an integrated, industry-standard stack.
  • Process Engineering: Defined branching, PR, and verification policies balancing speed and quality.
  • Team Architecture: Organized 9 developers into 3 functional subteams with clear leadership paths.

Professional Skills (soft skills)

  • Agile Facilitation: Led Sprint 1 planning and Scrum Master rotation for shared ownership.
  • Stakeholder Management: Kept kick-off with CEO/CTO focused on roles and deliverables.
  • Consensus Building: Advocated successfully for GitHub policies and Jira adoption.
  • Proactive Coordination: Designed role-rotation schedule to build versatility and prevent burnout.
